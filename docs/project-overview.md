## Purpose

JClaw is Abundent's **AI-powered automation platform**: a long-running Java service that hosts conversational LLM agents, schedules background tasks, executes tools (shell, filesystem, web, browser), and exposes a unified Web/Slack/Telegram/WhatsApp chat surface. It replaces two predecessor projects (`OpenClaw` Node.js, `JavaClaw` Spring Boot) with a single Java-first runtime on Play 1.x + a Nuxt 3 SPA.

Current version: **v0.7.18** (status: v0.1.0-alpha / work in progress per CLAUDE.md).

## Repository type

**Multi-part**. One git repo, two runtimes that ship as a single deploy unit (the Nuxt SPA is generated and copied into `public/spa/` of the Play dist zip).

| Part | Root | Type | Runtime |

### Core backend flows

- **Chat (web)** вҖ” `ApiChatController` вҶ’ `AgentRunner.run/runStreaming` вҶ’ `LlmProvider` + tool loop вҶ’ persists `Message` rows вҶ’ SSE to browser via `ApiEventsController`.
- **Channel inbound** вҖ” `WebhookSlackController` / `WebhookTelegramController` / `WebhookWhatsAppController` вҶ’ `AgentRouter.resolve` (3-tier: peer вҶ’ channel вҶ’ main agent) вҶ’ `AgentRunner.run` вҶ’ outbound via `channels.Channel.resolve()`.
- **Tasks** вҖ” `ApiTasksController` creates rows in `task`; `TaskPollerJob @Every("30s")` claims `PENDING` tasks whose `next_run_at вүӨ now`, runs each in a virtual-thread executor, handles CRON re-scheduling and exponential-backoff retries.
- **Skills** вҖ” files-on-disk skill definitions; `SkillLoader` + `SkillPromotionService` promote authored skills into the global registry. Promoted skills declare `commands:` that get persisted into `agent_skill_allowed_tool` for shell-exec allowlisting (the DB вҖ” not the workspace file вҖ” is authoritative).

### Frontend

Nuxt 3 SPA with `ssr: false`. Single global layout (`layouts/default.vue`), global auth middleware (`middleware/auth.global.ts`) that redirects to `/login` when `/api/config` 4xxs. All backend access through `$fetch` / `useApi` / `useApiMutation`. An `EventSource` connection to `/api/events` provides server-push notifications (e.g. `skill.promoted`, streaming tokens).
